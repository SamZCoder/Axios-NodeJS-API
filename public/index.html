NodeJs REST API


This Project is based on NodeJS and been created as a demonstration and material for 'Work with Axios Like a Pro' Courses.
As a Documentation we will be listing API rules and Endpoints as Reference.

 MongoDB Atlas  JWT Portal  Axios Repository

 Getting Started

  1. All end points prefixed with /api so for example you can get books as GET /api/books.
  2. Server Port can be Configured but set SERVER_PORT in .env file
  3. MongoDB Connection URI can be set as DB_URI in .env file
  4. JWT Encryption Secret can be set as JWT_SECRET in .env file

Available EndPoints

  1. Books /api/books
  2. Messages /api/messages
  3. Posts /api/posts
  4. Users /api/user

 Books EndPoints :

  • Get List of Books
    GET /api/books you can also limit the number of books returned but adding query string ?limit=LIMIT for example GET /api/books?limit=2
  • Get a Specific Book Record
    GET /api/books/BOOK_ID IDs are ranged for books from 1-5, for example GET /api/books/1

 Messages EndPoints :

  • Get List of Messages
    GET /api/messages
  • Create a new Message
    POST /api/messages

    Expected Payload

    {
  name : String, 
  subject : String, 
  content : String
}

 Posts EndPoints :

  • Get List of Posts
    GET /api/posts
  • Create a new Post (Auth Required)
    POST /api/posts

    Expected Payload

    {
  title : String, //Optional
  content : String,
  thumbnail : FILE //Optional
}
  • Get a Specific Post Record
    GET /api/posts/POST_ID IDs are Generated Automatically in Database, for example GET /api/posts/5e82696d8431b6281c16f761
  • Delete a Specific Post Record (Auth Required)
    DELETE /api/posts/POST_ID IDs are Generated Automatically in Database, for example DELETE /api/posts/5e82696d8431b6281c16f761
  • Update Existing Post (Auth Required)
    PUT /api/posts/POST_ID

    Expected Payload

    {
  title : String, //Optional
  content : String, //Optional
  thumbnail : FILE //Optional
}

 Users EndPoints :

  • Get List of Users
    GET /api/users
  • Register New User
    POST /api/users/register

    Expected Payload

    {
  name : String, //Optional
  email : String,
  password : FILE
}
  • Login and Authorize
    POST /api/users/login

    Expected Payload

    {
  email : String,
  password : String,
}
